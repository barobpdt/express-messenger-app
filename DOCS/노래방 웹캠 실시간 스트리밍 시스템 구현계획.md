# 노래방 웹캠 실시간 스트리밍 시스템

MediaSource API + WebSocket을 이용해 웹캠 영상을 실시간 스트리밍하고, 서버에서 녹화 후 다운로드할 수 있는 시스템을 구현합니다.

## 전체 흐름

```
[music_select.html]         [server.js]           [karaoke.html]
   getUserMedia()       →   WS binary relay   →   MediaSource API
   MediaRecorder           녹화 저장 (webm)        영상 재생
   선곡 정보 전송           REST API               곡 정보 + 녹화 목록
```

**포트 구분**
- Express/WS: 기존 서버 포트 (`.env`의 PORT) 그대로 사용
- 별도 서버 추가 없음

WebSocket `message` 핸들러에서 **binary 데이터** 처리 분기 추가:
- `ws.isBinary` 또는 `Buffer.isBuffer(message)` 판별

- 동시에 서버의 녹화 파일(`recordings/<session_id>.webm`)에 `fs.appendFileSync`로 저장

REST 엔드포인트 신규 추가:
```
POST   /api/karaoke/stream/start   → 세션 ID 발급, 녹화 파일 초기화
POST   /api/karaoke/stream/stop    → 녹화 파일 닫기
GET    /api/karaoke/recordings     → 녹화 파일 목록 (이름, 크기, 날짜)
GET    /api/karaoke/download/:file → 파일 다운로드 (Content-Disposition)
DELETE /api/karaoke/recordings/:file → 파일 삭제
```

WS 메시지 타입 약속 (JSON):
```json
{ "type": "karaoke-join",  "role": "viewer"|"streamer" }
{ "type": "karaoke-song",  "song": { "title", "artist", "number" } }
{ "type": "karaoke-start", "sessionId": "..." }
{ "type": "karaoke-stop" }
```

### 노래방 화면

#### [NEW] [karaoke.html](file:///c:/bpdt/project/express-sample/public/karaoke.html)

WebSocket으로 binary 청크 수신 → MediaSource API로 `<video>` 재생:
```js
const ms = new MediaSource()
video.src = URL.createObjectURL(ms)
ms.addEventListener('sourceopen', () => {
  const sb = ms.addSourceBuffer('video/webm;codecs=vp8,opus')
  ws.onmessage = e => {
    if (e.data instanceof Blob)
      e.data.arrayBuffer().then(buf => sb.appendBuffer(buf))
  }
})
```

레이아웃:
- 왼쪽: 웹캠 영상 (`<video>`) + 선택된 곡 정보
- 오른쪽: 녹화 목록 패널 (다운로드/삭제 버튼)

## User Review Required

> [!IMPORTANT]
> **녹화 저장 위치**: `프로젝트루트/recordings/` 폴더에 `.webm` 파일로 저장됩니다. 이미 `videos/` 폴더가 있는데, 기존 `videos/` 폴더를 사용할까요, 아니면 별도 `recordings/` 폴더를 만들까요?

> [!IMPORTANT]
> **인증 없음**: 스트리밍/녹화 API는 현재 인증 없이 로컬 네트워크 모든 사용자가 접근 가능합니다. JWT 인증이 필요하면 알려주세요.

> [!NOTE]
> **코덱 호환성**: `video/webm;codecs=vp8,opus`는 Chrome/Edge에서 동작합니다. Safari는 미지원이나, 로컬 노래방이므로 무방합니다.
