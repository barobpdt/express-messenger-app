# Mobile Geolocation Sharing Implementation Plan

This plan outlines the steps to implement a feature allowing users to share their current location, which will be displayed on a map for other users in the chat.

## Proposed Approach

We will use the browser's native `navigator.geolocation` API, which works on both desktop browsers and mobile devices (via web views or direct browser access). To render the map without requiring a paid API key (like Google Maps), we will use **Leaflet.js** with **OpenStreetMap** tiles, which is free, open-source, and easy to embed.

### 1. Backend (`server.js`)
Location sharing is just another message type over the existing WebSocket connection.
- No database schema changes are strictly necessary unless we want to persist location history (currently, messages are ephemeral in memory/not saved).
- Update the WebSocket message handler to support `type: 'location'`.
- Pass through the `lat` (latitude) and `lng` (longitude) coordinates to the target user or broadcast to all.
- Update FCM push notifications to support the `location` message type (e.g., body: "📍 위치를 공유했습니다.").

### 2. Frontend (`messenger.html`)
- **[MODIFY] `messenger.html`**
    - **Libraries**: Inject Leaflet.js CSS and JS via CDN in the `<head>`.
    - **UI**: Add a "Location Share" button (📍) next to the file upload button in the input area.
    - **Logic**:
        - Implement `shareLocation()` using `navigator.geolocation.getCurrentPosition()`.
        - Send a WebSocket payload: `{ type: 'location', lat: ..., lng: ..., targetUser: ... }`.
    - **Rendering**:
        - Update `appendMessageBubble()` to handle `data.type === 'location'`.
        - When rendering a location, dynamically generate a `<div>` for the map, initialize a Leaflet map object (`L.map`), and place a marker at the given coordinates.

## User Review Required
> [!NOTE]
> For the map renderer, **Leaflet + OpenStreetMap** is proposed because it is completely free and requires no API keys, which allows immediate implementation. If you specifically require **Google Maps**, a Google Maps API Key will be needed. Please confirm if OpenStreetMap is acceptable!

## Verification Plan
1. Send a location message from the browser.
2. Verify the browser prompts for location permission.
3. Verify the WebSocket broadcasts the `lat`/`lng`.
4. Verify the Leaflet map renders correctly inside the chat bubble on the receiving end with a marker pointing to the location.
