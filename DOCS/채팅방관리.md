# 채팅방 CRUD 구현 계획

채팅방 관련 정보를 데이터베이스에 저장하고, 이를 관리할 수 있는 RESTful API와 클라이언트 테스트 화면을 구현합니다.

## Proposed Changes

### Database Schema
#### [MODIFY] [schema.js](file:///c:/bpdt/project/express-sample/db/schema.js)
Drizzle ORM을 사용하여 다음 2개의 테이블을 추가합니다.
- `chatRoomsTable` (chat_rooms): `id`, `name`, `description`, `owner`, `createdAt`
- `chatRoomUsersTable` (chat_room_users): `id`, `roomId`, `username`, `joinedAt`

### Server Backend
#### [MODIFY] [server.js](file:///c:/bpdt/project/express-sample/server.js)
새로 만든 테이블 스키마를 import 하고, 아래와 같은 CRUD API 엔드포인트를 구현합니다.
- `GET /api/chat-rooms`: 채팅방 목록 조회
- `POST /api/chat-rooms`: 새 채팅방 생성 (성성 시 방장 자동 참여)
- `PUT /api/chat-rooms/:id`: 채팅방 정보(이름, 설명) 수정
- `DELETE /api/chat-rooms/:id`: 채팅방 삭제 (해당 방의 참여자 목록도 연쇄 삭제)
- `GET /api/chat-rooms/:id/users`: 특정 채팅방의 참여자 목록 조회
- `POST /api/chat-rooms/:id/users`: 채팅방 입장 (참여자 추가)
- `DELETE /api/chat-rooms/:id/users/:username`: 채팅방 퇴장 (참여자 제거)

### Frontend Sample Page
#### [NEW] [chat_rooms.html](file:///c:/bpdt/project/express-sample/public/chat_rooms.html)
테스트를 위한 간단한 CRUD 샘플 페이지를 생성합니다.
- 채팅방 생성 폼
- 채팅방 목록 테이블
- 각 채팅방별 수정/삭제, 입장/퇴장 테스트 버튼 포함
- JS `fetch`를 이용하여 서버의 API와 통신하는 로직 구현

### Manual Verification

2. 브라우저에서 `http://localhost:8081/chat_rooms.html`에 접속합니다.
3. [채팅방 생성] 기능이 정상 동작하는지 확인합니다.
4. 방 목록이 올바르게 출력되는지 확인합니다.
5. 특정 방의 정보를 [수정]할 수 있는지 확인합니다.
6. [입장] 버튼을 통해 사용자가 방에 추가되고, 참여자 목록을 볼 수 있는지 확인합니다.
7. [삭제] 버튼을 눌러 채팅방이 정상 제거되는지 점검합니다.
