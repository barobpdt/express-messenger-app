# 태블릿 주문 시스템 (order.html) 구현 완료

새로운 태블릿 주문 어플리케이션(`order.html`)의 모든 핵심 기능(Phase 1 ~ Phase 5) 구현을 완료했습니다.

### 1. 세련된 장바구니 및 알림 팝업 (Phase 2)
주문을 전송할 때 사용하는 알림창을 기존 브라우저 기본 알림(`alert`)에서 **프리미엄 다크 느낌의 커스텀 팝업 레이어**로 변경하여, 디자인 완성도를 획기적으로 높였습니다. 장바구니 내용은 `[POST] /api/order/orders` API를 통해 현재 테이블 번호(`tableId`)와 함께 데이터베이스에 안전하게 기록됩니다.

### 2. 누적 주문 내역 (빌지) 확인 기능 (Phase 3)
화면 왼쪽 아래의 **[🧾 주문내역]** 버튼을 터치하면, 현재 테이블(`tableId`)에 진행 중이거나 제공 완료된(단, 결제는 안된) 주문들을 모두 합산하여 보여주는 **영수증 모달** 기능이 도입되었습니다. (동일한 메뉴는 수량과 금액이 자동으로 합산됩니다.)

### 3. 직원 호출 및 카운터 실시간 알림 기능 (Phase 4)
**[🔔 직원 호출]** 메뉴를 통해 고객이 탭 화면에서 "물", "냅킨", "소주잔" 등 자주 찾는 항목들을 터치 한 번으로 호출할 수 있습니다.
호출 즉시 WebSocket을 통해 카운터용 **주문 관리 페이지 (`order-management.html`) 우측 상단에 보라색 알림(Toast)과 알림음**이 함께 뜹니다.

### 4. 대기 화면(Screensaver) 비디오 재생 (Phase 5)
태블릿에서 30초(시연용, 실제 적용 시 늘릴 수 있습니다) 동안 터치 이벤트가 발생하지 않으면, **먹음직스러운 식욕 자극 조리 영상이 전체화면으로 자동 재생**됩니다. 고객이 화면을 한 번이라도 터치하면 즉시 주문 메인 창으로 복귀합니다. 이 때 만약 빈 테이블 상태였다면, 이전에 남아있던 장바구니 내용물과 모달 창도 자동으로 초기화되어 다음 손님을 맞이할 준비를 마칩니다.
